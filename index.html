1 use transaction

in ssms, when using t-sql to manipulate records, use "begin transaction" plus "rollback" before "commit".
e.g.
begin transaction
delete from tablename where id > 100
rollback
and ssms will tell you how many rows are affected.
if you feel that's correct, change rollback to commit, and execute
p.s. this solution may not be exactly correct, it works for me so far, I did not study into the implications yet.

1 clone a database

make a clone of a database within the same sql server instance:

  1. in the sql server instance, create an empty new database, name it as originame_clone
  2. back up the database I want to clone, note the path backup file is kept, name it as origin_bk_path
  3. right click the database: originame_clone, go to tasks --> restore
  4. set device to origin_bk_path
  5. under select a page, choose: File
  6. the origin file path and destination file path are set by default, change the path of the destination file to what you want
  7. apply

there is also a case whereby I want to clone a database from one sql server instance to another, or even from one sql server to another sql server.
That is not discussed here, for quick reference, it's said by MSDN that I can do that via "Copy Database" option via right click a database --> Tasks --> Copy database

fix the width of my web pages

it's a good practice to make sure the text doesn't trancate when fold the web page to the side of the screen. howerver, it's very annoying to write a lot of br tags to change-line manually. Now here is the elegant solution:

  1. enclosing the whole web page body in a div
  2. set the width attribute of that div to 700px
    3. code examples:
    div.content {width: 700px}
    <body>
    < div class="content">
    ...
    </div>
    </body>
